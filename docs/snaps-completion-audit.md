# snaps Completion Audit

This audit tracks the current snaps conversion state without requiring dependency installation or a live runtime.

## Objective

Keep Postiz as the SNS publishing infrastructure and add snaps-specific layers for:

- Ollama-based AI content transformation
- RAG-style platform examples
- Korean and China SNS support
- Pixelle short-form video workflow
- AI analytics reports
- unified feedback management
- snaps-branded product/deployment surface

Concrete success criteria for the skipped-install pass:

- Do not wait on dependency installation.
- Keep the existing Postiz OAuth, media, scheduling, organization, and provider core in place.
- Add snaps-specific backend, frontend, AI, RAG, analytics, video, Korean and China SNS, and feedback artifacts.
- Provide local verification that does not require a running Postiz server.
- Leave the remaining runtime gates explicit instead of treating static checks as full product completion.
